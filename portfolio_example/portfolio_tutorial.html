Portfolio Template Tutorial

We will build our own HTML-template throughout this little tutorial. It may a ready to use template for all your pages in the your Portfolio. Therefor we go step by step and learn how to integrate each feature of the portfolio template.

Starting from Scratch

To create a web page from scratch means, we are going to use a new empty text file and integrate all relevant code snippets that form a template page. Name it "template.html" for example.

Use the same folder "portfolio" that contains all the files we need for the template and the pages your are going to create later!

The first code snippet is a minimal html5 code to create a page:

<!DOCTYPE html>
<html lang="en">
  <head>
    
    
    
    This is the Hello World example
  </head>
  <body>
    
    
Hello World!

  </body>
</html>

The example shows a line of text when opened in a browser. But there are far more lines in the code. On W3Schools HTML-Tutorial we can learn what they are for.

The next step is to define the structure of the document. For our portfolio we'll use the HTML elements article and section with headings and normal text paragraphs. And we put them into the body-tag (we want to replace the body part from the first code snippet).

In the next step we are adding CSS to format the HTML-structure. The Cascading Style Sheet is an essential part of a HTML-page. In our case we use water.css, a free CSS framework, which is a minimal CSS-formatting for HTML-elements, that can be loaded into our page and that can easily be expanded to needs of the portfolio.

Waters.css is loaded into a HTML page using the link-tag within the head-element.

Reload the page to see the changes. It looks much nicer now, doesn't it? There are CSS-styles for all HTML-elements to give the page a pleasant look, without writing complicated CSS-expressions.

By the way, the page is also responsive, means, it adapts to different screen sizes, for example on mobile devices.

OK, the last step for now, is to modify and extend waters.css. We need an additional CSS to do that, so let's create a new text file and call it template.css.

The CSS-rules we'll see below allow a broader content area, it will be set to 960px here. Also we want the content of the article to stand out from the rest of the browser window.

/* TEMPLATE.CSS */

:root {
    --background-body: #ddd; /* changed */
    --background-main: #fff; /* new */
}

article {
    background-color: var(--background-main);
    padding-top: 1em;
    padding-bottom: 1em;
    padding-left: 2em;
    padding-right: 2em;
}

waters.css defines also variables, that are defined in a :root CSS-rule. Here we overwrite and create new variables. You'll find more about CSS-variables on W3Schools.

A special "Authors" section

A good scientific report has an area to display authoring information and we want to use a section with horizontal positioned boxes. Because the default in displaying boxes in HTML and CSS is a vertical positioning, we have to use a more advanced CSS-technique and will try to be really minimal in the sense of collapsing HTML-elements. The solution for a horizontal layout here is to use the css-flex layout.

Imagine you want a centered box with an alternative background color to show your personal information inside the page, we can use a div-element and combine it with a css-class.

Put this into the HTML

and this into the CSS

    padding-left: 1em;
    padding-right: 1em;

Navigation Menu

If we want to present our content on multiple pages we need a navigation with links to the single pages. Often this is done by a menu, for example at the top of the page. The menu component I choosed, can be found on W3Schools.

I splitted the code into two files: w3menu.css and w3menu.js, so this time we not only have a CSS but some lines of javascript, that does some action when the menu adapts to smaller screens.

By the way: the menu uses a horizontal layout. But this time not using a flex- but a float-layout. This was the very first approach of CSS to present HTML boxed elements horizontally by literally forcing them to display inline, and "floating" with the text.

Again we need to link the needed files in the head of the document:

  
  
  <script src="w3menu.js">

And then we use some dummy links as the menu of the page:


    

        Home
        Topics
        About

This menu collapse on small screens and shows a hamburger-like menu icon: - The layout of the menu therefor changes and displays its items vertically.

Header with picture

The first page of Website often shows an image or a logo to catch the attention of the reader. In the portfolio template page it is a little abstract world map, that lays in background, behind the Website main title.

To achieve this, I grouped the menu and the title image into a HTML-header element. Inside the header the nav-tag is followed by a section with id "header", containing the headings (h1 and h2). This section with id=header is formatted by CSS to include an background image.

head and header are not the same in HTML. While head is an essential (but invisible) part of every HTML-document, header is an semantic page structuring element (in HTML5) to make document code more readable.

To achieve this, replace your old menu part with this new header block:

  <header>
        

            Home
            Topics
            About
            
                
        

        

        
Geovisualization

        
Portfolio Winter 2023/2024

        

  <header>

And the CSS looks like this:

#header {
    display: block;
    position: relative;
    background: url('./images/background-image.png');
    background-position: center;
    background-repeat: no-repeat;
    background-size: cover;
    height: 500px;
    padding-top: 1em;
    font-size: large;

}

#header h1, #header h2 {
    text-align: center !important;
    color: white;
    text-shadow: 1px 1px 2px rgba(150, 150, 150, 1);
}

#header h1 {
    font-size: 4em;
}

You can add the CSS-rules to the "template.css" file or into an extra CSS-file, if you like the component approach (means separating the different parts). In the latter case don't forget to link the CSS in the head-tag.

When you use a header tag as a container, and a section with the id "header", the background image will be loaded. To use a different image, try to change the settings for background: url, it should always contain the valid path to your image.

Code Highlighting

Code Highlighting is done by prism. It uses CSS and javascript to style a nested block of pre and code tags. Simply add the links in the head element and put the examples into the block. Use the class for your language in pre and let prism do the rest. Only for showing HTML-examples we need an additional javascript to "escape" the HTML relevant signs like < or >.

Put this into the head-element:

Add this somewhere in an article/section:

 
some html code

And put this script bevor the closing body tag:

    
    <script>
    document.querySelectorAll("code.language-markup, code.language-html")
        .forEach( 
            function(element) {
                element.innerHTML = element.innerHTML.replace(/</g, "&lt;").replace(/>/g, "&gt;").replace(/"/g, "&quot;").replace(/'/g, "&#039;");
            }
    );
    </script>

Inside the prism directory, you'll find a list with the available languages!

Citations

Citr.js is a tiny Javascript which searches a HTML-page (the DOM) for quotes.

In html itself we have two tags for citing:

  • <blockquote> can be used for direct quotes in form of a paragraph.
  • <q> can be used inside a paragraph, inline for a short or indirect quotes.

Both elements have a "cite" and a "title" attribute to specify the quoted source. A citation then looks like this:

 

In African societies, participation in music may be a voluntary activity 
or an obligation imposed by one's membership in a social group. Such a social group 
may be a descent group (a group of people who trace their ancestry back to the 
same person), or it may be any group based on the broader societal classifications 
of age, sex, interest or occupation.

This quote comes directly from the citr.js examples.html

Here title contains the bibliographic content and cite the (internet) source, where the title can be found.

Now citr.js uses the blockquote and q-elements as well as the attributes cite and title to create links and reference list at the end of an HTML-article.

Therefore you can use CSS-classes to get that automatic behavior. Citr.js knows the classes "footnote", "marginalia", "harvard" and "indirect".

Example direct blockquote

Change the blockquote from above by adding class="footnote":

 
    
In African societies, participation in music may be a voluntary activity 
    or an obligation imposed by one's membership in a social group. Such a social group 
    may be a descent group (a group of people who trace their ancestry back to the 
    same person), or it may be any group based on the broader societal classifications 
    of age, sex, interest or occupation.
J. H. Kwabena Nketia, 'The Music of Africa' (1974), p. 35

And also, to see the effect, link the citr.js at the bottom of our HTML-article tag:

    

    
    
      
      <script type="text/javascript" src="./cite/citr.js">

Added to the document, you'll see the reference/footnote that was created by citr.js.

Additionally we can add CSS-styles for the elements and classes. Waters.css adds CSS to the elements, that do not work well, in my opinion, so I put two CSS-rules for quotes into template.css to unset some of the waters.css formatting:

q {
	border-left: unset;
	margin: unset;
	padding: unset;
    font-style: unset;
}

blockquote p {
    margin: unset;
}

Overlays

Overlays are popup areas that, for example, open by clicking a link or a button and position thereself upon the text. They are often used to show images in a bigger size or presenting messages, like a progress bar. My Overlay component for the portfolio is based on a W3Schools example.

I tried it using HTML-sections and an iframe. With HTML-sections a part of HTML code is hidden by default and popups on request. Using an iframe, the overlay will load an additional HTML-Dokument!

My Overlay comes with two files overlay.js and overlay.css, which can be linked to the HTML-head part:

    
    
    <script src="overlay.js">

The CSS-part deals with the positioning and display of the overlay content. Every overlay defines a class "overlay" for the allover frame (e.g. a div or a section) and uses an individual id to be switched on and of by the javascript functions "ov_on" and "ov_off". The inner content frame (e.g. a div) uses the class "ovelem" (the overlay element). Inside some element should be clickable and execute ov_off. Tips

  • readability and structure: If you write a HTML-document your main article should contain a h1-heading. Create sections for each h2-heading.
  • use indentations in nested HTML-elements! (Only in pre-code boxes for code highlighting line breaks and indentations will be visible, but not in the normal display)
  • if you use sub-directories for pages use correct relative links to the parent directories and files (e.g. ../images/background-image.png)
