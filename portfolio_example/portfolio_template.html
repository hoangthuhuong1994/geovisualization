HTML

This HTML document uses semantic elements where ever it is possible. Semantic elements are used to structure the documents in meaningful parts and to improve the readability of the code. So you'll find

  • a <header>,
  • a <nav>(igation),
  • an <article> and
  • some <sections>

Each of the parts has a predefined format, a css-style, sometimes there are variants of the styling by using css-selectors and attributes (class or id). But mainly it was tried to avoid too many styling variants, so the HTML-code stays clean and we can concentrate on the content, not so much on the formatting.

Water.css

The basis of the layout and formatting for Portfolio-Template is Water.css, you'll find the source on github.

Water.css is a drop-in collection of CSS styles to make simple websites like this just a little bit nicer.

There are preformatted styles for most of the HTML-tags. There is an additional css file included into this page: main.css. It is based on water.css but defines divergend and new styles for the portfolio. Below you'll find some examples of HTML-elements and how they look.

Tip: Use the inspection tool of your browser to see the underlaying code and css definitions. In Firefox this is a right click on the part you want to explore and then press 'Q' on the keyboard to open a window with the development tools.
